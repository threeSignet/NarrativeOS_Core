Findings
- relatedEntity 过滤使用 JSON LIKE 匹配而非 json_each，在极端场景（entity ID 是另一个 ID 子串）可能误匹配，当前数据规模可接受，后续可优化
- addMilestone 不校验状态转换合法性（如从 FILLED 回退到 PLANTED），此校验应在 ThreadResolver 层实现
- ThreadStore 的 SQL 字段过滤（direction/type/severity/status）直接映射为 WHERE 条件，JSON 字段过滤（relatedEntity/arcTag/excludeArcTags）使用 LIKE 或直接列匹配
