# 关系创建前端闭环 + 隐藏 bug 修复

## 架构定位
本次开发在第④层（BFF 路由）+ 第⑤层（前端 UI），不碰第①层 Core。
第③层 RelationService 有 3 个 bug 要顺带修（不改变接口契约，只补收敛逻辑）。

## 体验设计：一步创建 + 待确认
用户操作极简（和实体里程碑③对称）：
1. 点"新建关系" → 选源实体 + 目标实体 + 关系类型 → 创建
2. 后端自动走：createRelationCandidate → submitRelationCandidate（沙盒推演+PV+PendingDecision）
3. 侧栏"待确认"面板出现"确认关系：X - 兄妹 → Y"
4. 点"确认" → confirmRelationCommit → 写进 Core → 图谱出现 committed 边

## 后端 bug 修复（3处，第③层）

### Bug 1：RelationService 缺 listRelationHints
- **文件**：`src/writing/services/relation-service.ts`
- **改法**：加 `listRelationHints(ctx, filter?)` 方法，转发到 `store.listRelationHints`（store 已有，行 2661）
- **影响**：前端能列出 Agent 提取的关系提示

### Bug 2：confirmRelationCommit 不 resolve PendingDecision
- **文件**：`src/writing/services/relation-service.ts:279-319`
- **改法**：在 confirmRelationCommit 成功分支（result.success）末尾，加 `this.workflow.resolvePendingDecision(ctx, decisionId, {status:'resolved', note: coreEventId})`
- **关键**：要从 submitRelationCandidate 建的 PendingDecision 里找到 linkedObjectId===pv.id 的 open 决策来 resolve
- **影响**：待确认面板不再有悬挂的 open 决策

### Bug 3：confirmRelationCommit 的 coreRefs.predicate 语义错位
- **文件**：`src/writing/services/relation-service.ts:306`
- **现状**：predicate 字段塞了 `pv.coreProposalId`（proposal ID），不是真正的关系谓词
- **改法**：predicate 从 candidate 的 relationTypeId 取（或从 PV 的 simulationInputs.factChanges[0].predicate 取）
- **影响**：coreRefs 语义正确，图谱能正确显示关系类型

## BFF 路由（第④层，扩展 relations.ts）

当前 `apps/bff/src/routes/relations.ts` 只有 GET graph。新增写入路由：

1. **GET /api/projects/:pid/relation-hints** → listRelationHints（Bug1修完后）
2. **POST /api/projects/:pid/relation-hints/:id/confirm** → confirmHintToCandidate
3. **POST /api/projects/:pid/relation-hints/:id/ignore** → ignoreHint
4. **POST /api/projects/:pid/relations** → 一步创建：createRelationCandidate + submitRelationCandidate（前端"新建关系"按钮调这个）
5. **POST /api/projects/:pid/relations/:id/confirm** → confirmRelationCommit（写进 Core）
6. **POST /api/projects/:pid/relations/:id/deprecate** → deprecateRelationCandidate
7. **POST /api/projects/:pid/associations** → createAssociation（创作关联，不进 Core）
8. **DELETE /api/projects/:pid/associations/:id** → archiveAssociation

扩展 decisions.ts 的 resolve，处理 confirm_proposal（当前只处理 confirm_entity）：
9. **decisions.ts** 加 confirm_proposal 分支：resolve 时调 confirmRelationCommit
