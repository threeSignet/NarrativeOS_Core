# 基建阶段:共享组件层 + 现有插件迁移统一

## 总目标与工作流闭环

建立 `src/components/` 共享组件库(Ui 前缀),把现有 4 个插件从"各写各的重复 UI"迁移到统一组件,顺带把 AgentPanel 从 App.vue 硬编码规范化为可插拔面板。

每个节点完成后,我产出**自审报告**(对照本计划清单逐项打勾 + 记录偏差/技术债),你在浏览器验证通过后,我进入下一节点。4 个插件全部迁移完 = 阶段完成。

```
节点 0 (契约) → 节点 1 (Ui 基础组件) → 节点 2-5 (逐插件迁移+验证)
                                         ↓
                              节点 6 (统一性打磨 + 最终自审)
```

## 架构定位

- **只动前端 ⑤层**:不碰 BFF/Core/service。本阶段零后端改动。
- **只抽展示型 + 轻交互组件**:共享组件**不带业务逻辑**,纯 props/emit;业务逻辑留在插件 + store。
- **强制依赖约束**(从 stores/composables 调查得出):
  - 写操作走 `store.action(projectId, ...)`,组件**绝不**直接调 `api/*`
  - 提示走 `useToast()`、确认走 `useConfirm()`、偏好走 `usePreferences()`、切项目走 `useProjectSwitch()`——**不允许**共享组件内自造这些
  - 样式**只引用** `tokens.css` 的 CSS 变量(`--sp-*/--fs-*/--r-*/--text-*/--bg-*/--border*`),禁止硬编码色值

## 节点 1:Ui 基础组件实现

**目标**:把 4 个插件里重复的 UI 模式抽成 Ui 组件,API 设计对齐现有用法。**此节点不迁移任何插件**,只产出组件 + 自审 API 设计。

### 1.1 base/UiButton.vue(最高优先,统一 7 套按钮类)
收敛现状散落的 7 套按钮:`.btn`/`.icon-btn`/`.op-btn`/`.tb-btn`/`.doc-action-btn`/`.filter-chip`/`.layer-chip`。
- **Props**:
  - `variant: 'default' | 'primary' | 'ghost' | 'danger'`(替代 .btn--primary/.btn--ghost/.op--primary/.op--ghost)
  - `size: 'sm' | 'md'`(替代 .btn--sm)
  - `icon?: boolean`(true=纯图标按钮,方形,替代 .icon-btn/.tb-btn/.doc-action-btn)
  - `disabled?: boolean`
- **Slots**:default(文字/图标)
- **Emits**:click
- 内部走全局 `.btn` 基类 + variant 组合 class

### 1.2 base/UiIcon.vue
统一现状散落的 inline SVG 写法。每个插件手写 `<svg viewBox=... stroke=...>`。
- **Props**:`name: string`(从预定义图标集查)、`size?: number`(默认 18)
- 内置图标集(从现有插件提取):folder/folder-open/file/plus/import/export/copy/search/trash/chevron-down/chevron-right/close/refresh/bold/italic/...
- 用 `<component :is>` 渲染内部 SVG path 组件

### 1.3 base/UiEmpty.vue(统一 4 处 empty-state)
- **Props**:`title?: string`、`description?: string`、`icon?: string`(传给 UiIcon)
- **Slots**:default(自定义内容,如带操作按钮的空状态)、icon(自定义图标)
- 自适应:在侧栏内(高度自适应) vs 主区(撑满)
- 收敛现状 4 处 `<div class="empty-state">` 手写 + 内联 style 微调

### 1.4 base/UiStatusDot.vue(统一 status-dot 双轨冲突)
现状:shell.css 全局有 `.status-dot--committed` 等,EntityGraphSideView 又 scoped 重写 `.sd--*`——双轨冲突。
- **Props**:`status: string`(实体状态枚举)、`color?: string`(自定义颜色,用于来源层)
- 内部用一个 status→color 映射表(单源),消除双轨

### 1.5 layout/UiSideHead.vue(统一 3 处 side-head)
- **Props**:`title: string`
- **Slots**:actions(右侧按钮区)
- 收敛现状 3 处 `<div class="side-head"><span class="side-title">...<div class="side-actions">`

### 1.6 layout/UiSearchBar.vue(统一 2 处 side-search)
- **Props**:`modelValue: string`(v-model)、`placeholder?: string`
- 内置搜索图标 + 输入框,聚焦态 border 变色
- 收敛 DocumentTreeView/EntityGraphSideView 各写的 search 结构

### 1.7 layout/UiSideBody.vue
- **Slots**:default
- 纯样式容器:flex:1 + overflow-y:auto,收敛现状各插件 `<div class="side-body">` 重复

### 1.8 layout/UiPanelFooter.vue
- **Slots**:default
- 侧栏底部固定区(如待确认决策面板),border-top + 背景色
- 收敛 EntityGraphSideView 的 decisions-panel

### 1.9 form/UiInput.vue / UiSelect.vue / UiTextArea.vue
- **Props**:标准 v-model + disabled + placeholder
- 收敛 create-form 内的 .form-input 三种控件

### 1.10 form/UiInlineForm.vue

### 1.11 feedback/UiBadge.vue / UiChip.vue
